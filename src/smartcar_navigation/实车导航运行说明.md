# 实车导航运行说明

本说明补齐 LAB10 在真实 LIMO 小车上的导航入口。

当前工作区原本只有仿真导航代码，没有单独保留“实车导航启动层”。因此这里新增的是一套通用实车导航 launch 文件，复用同一份地图、AMCL、`move_base` 和规划参数，但把真实车最容易变化的接口都改成了可配置参数。

## 适用前提

启动本导航之前，真实 LIMO 底层应已经满足以下条件：

- 能发布激光数据，例如 `/scan` 或 `/limo/scan`
- 能发布里程计 `odom`
- 能提供 `map -> odom -> base_footprint` 或兼容的 TF 链
- 能接收速度指令 `/cmd_vel`

如果你的实车底层话题名和默认值不同，直接在 `roslaunch` 命令后覆盖参数即可。

## 新增的实车导航入口

- `limo_real_amcl.launch`
- `limo_real_navigation_dwa.launch`
- `limo_real_navigation_teb.launch`
- `limo_real_navigation_teb_astar.launch`

## 默认接口

- 激光话题：`/limo/scan`
- 激光坐标系：`laser_link`
- 里程计话题：`odom`
- 里程计坐标系：`odom`
- 机器人底盘坐标系：`base_footprint`
- 速度控制话题：`/cmd_vel`
- 地图坐标系：`map`

## 运行方式

### 1. 编译并加载环境

```bash
cd ~/myfile/robote_project/limo_ws
catkin_make
source devel/setup.bash
```

### 2. 启动真实车底层驱动

底层驱动不在本次补齐范围内，但应先保证雷达、里程计和 TF 正常。

建议先检查：

```bash
rostopic list
rostopic echo /odom
rostopic echo /scan
rosrun tf view_frames
```

### 3. 启动定位或导航

仅定位：

```bash
roslaunch smartcar_navigation limo_real_amcl.launch
```

默认全局规划器 + DWA：

```bash
roslaunch smartcar_navigation limo_real_navigation_dwa.launch
```

默认全局规划器 + TEB：

```bash
roslaunch smartcar_navigation limo_real_navigation_teb.launch
```

A* 全局规划器 + TEB：

```bash
roslaunch smartcar_navigation limo_real_navigation_teb_astar.launch
```

### 4. 如果实车话题名不同

如果你的真实车驱动发布的是 `/scan` 而不是 `/limo/scan`，则可以这样启动：

```bash
roslaunch smartcar_navigation limo_real_navigation_teb.launch \
  scan_topic:=/scan \
  sensor_frame:=laser_link \
  odom_topic:=odom \
  odom_frame:=odom \
  base_frame:=base_footprint \
  cmd_vel_topic:=/cmd_vel
```

## 关键设计

与仿真版相比，实车版主要增加了这些可配置项：

- `scan_topic`
- `sensor_frame`
- `cmd_vel_topic`
- `odom_topic`
- `odom_frame`
- `base_frame`
- `global_frame`

这样即使真实车底层代码缺失，只要重新接回驱动并保证导航常用接口存在，就能直接复用这套实验代码。
