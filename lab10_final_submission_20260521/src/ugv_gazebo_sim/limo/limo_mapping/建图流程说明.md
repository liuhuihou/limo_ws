# limo_mapping

LIMO 的建图流程包，负责把激光数据接到 SLAM 后端并输出地图。

## 功能说明

这个包实现的是“建图流程”，不是底层传感器驱动。它在已有的 LIMO Gazebo 仿真基础上完成：

- 订阅 `/limo/scan` 激光数据
- 使用 `odom` 和 TF 做位姿估计辅助
- 运行 SLAM 后端生成 `/map`
- 将建好的地图保存为 `map.yaml` 和 `map.pgm`

当前工作区保留了三个建图入口：

- `gmapping.launch`
- `hector.launch`
- `cartographer.launch`

其中 `cartographer.launch` 在这个工作区里是兼容入口，实际回退到 `gmapping.launch`，因为当前环境没有集成 `cartographer_ros`。

## 运行流程

### 1. 编译工作区

```bash
cd ~/myfile/robote_project/limo_ws
catkin_make
source devel/setup.bash
```

### 2. 启动仿真

```bash
roslaunch limo_gazebo_sim limo_four_diff.launch
```

### 3. 启动一种建图方式

#### Gmapping

```bash
roslaunch limo_mapping gmapping.launch
```

#### Hector

```bash
roslaunch limo_mapping hector.launch
```

如果系统没有安装 Hector：

```bash
sudo apt-get install ros-noetic-hector-mapping
```

#### Cartographer 兼容入口

```bash
roslaunch limo_mapping cartographer.launch
```

说明：

- 这个入口为了保留实验流程命名。
- 当前工作区没有真正启用 `cartographer_ros`。
- 启动后实际走的是 `gmapping`。

### 4. 键盘控制小车移动完成建图

```bash
rosrun limo_gazebo_sim keyboard_teleop.py
```

### 5. 保存地图

```bash
mkdir -p ~/myfile/robote_project/limo_ws/map
rosrun map_server map_saver -f ~/myfile/robote_project/limo_ws/map/map
```

保存后会得到：

- `~/myfile/robote_project/limo_ws/map/map.yaml`
- `~/myfile/robote_project/limo_ws/map/map.pgm`

## 文件作用

`launch/gmapping.launch`

- 使用 `slam_gmapping` 建图
- 适合当前工作区默认流程

`launch/hector.launch`

- 使用 `hector_mapping` 建图
- 对激光数据和 TF 较敏感

`launch/cartographer.launch`

- 建图兼容入口
- 当前工作区中回退到 `gmapping.launch`

`package.xml`

- 声明建图流程依赖

`CMakeLists.txt`

- 安装 launch 和 README 文件
